## Metodología de Trabajo

### Filosofía Neero

**1. Agile Adaptado (No Dogmático)**
- Sprints 2 semanas (flexibles según proyecto)
- Demos semanales con cliente (feedback continuo)
- Backlog priorizado por valor negocio (no técnico)

**2. Client Co-Creation**
- Cliente participa diseño (no solo recibe producto final)
- Usuarios piloto desde semana 1 (Liced en caso Contecsa)
- Ajustes durante desarrollo (no solo UAT final)

**3. Documentation-First**
- Antes de escribir código, documentar arquitectura
- Manual usuario en paralelo a desarrollo (no después)
- Transferencia conocimiento continua (no dump final)

**4. Quality Gates**
- Code review 100% código (pair programming críticos)
- Testing automatizado desde día 1
- UAT formal cada milestone (criterios objetivos)
